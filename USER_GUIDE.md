# APIDoc 사용자 가이드

이 문서는 APIDoc의 주요 기능과 사용 방법을 안내합니다.

---

## 1. APIDoc 소개 및 기술 스택

### 🎯 개발 배경
여러 곳에 흩어져 있는 API 문서를 하나의 페이지에서 통합 관리하고, 문서의 변경 이력을 체계적으로 추적하여 개발 생산성과 협업 효율성을 높이기 위해 만들어졌습니다.

### 💻 주요 기술 스택 (Tech Stack)
-   **Backend:** Node.js, Express.js
-   **Database:** MongoDB
-   **Frontend:** HTML, CSS, JavaScript (Vanilla)
-   **기타:**
    -   `json-diff`: API 버전 간의 차이점 비교
    -   `axios`: Swagger/OpenAPI 데이터 요청
    -   `helmet`, `cors`, `morgan`: 보안 및 서버 로깅

---

## 2. 사용 방법 (1): API 문서 등록 및 조회

새로운 API 문서를 시스템에 등록하고 상세 내용을 확인하는 방법입니다.

1.  **➕ API 등록 시작**
    -   메인 페이지 우측 상단의 `+` 아이콘을 클릭합니다.

2.  **🔗 Swagger/OpenAPI URL 입력**
    -   팝업창에 관리할 API의 `Swagger.json` 또는 `OpenAPI.json` 파일의 URL을 입력하고 'Import' 버튼을 클릭합니다.

3.  **📋 목록에서 확인**
    -   잠시 후, 메인 페이지의 API 목록에 방금 등록한 문서가 나타납니다.

4.  **📄 상세 정보 조회**
    -   목록에서 해당 API를 클릭하면, 엔드포인트, 모델, 인증 방법 등 상세 정보가 담긴 페이지로 이동합니다. 'History' 탭에서는 이 API의 변경 이력을 확인할 수 있습니다.

---

## 3. 사용 방법 (2): API 변경 내용 추적 및 비교

기존 API 문서의 변경 사항을 추적하고, 이전 버전과 비교하는 방법입니다.

1.  **🔄 새 버전 가져오기 (Import New Version)**
    -   API 상세 페이지에서 'Import New Version' 버튼을 클릭하여 변경된 `Swagger/OpenAPI` URL을 다시 가져옵니다. 시스템은 이를 새 버전으로 인식하고 저장합니다.

2.  **⏳ 비교할 버전 선택**
    -   'History' 탭으로 이동하면 새로 추가된 버전을 포함한 전체 이력 목록이 나타납니다.
    -   목록에서 비교하고 싶은 두 개의 버전을 선택한 후, 'Compare Selected Versions' 버튼을 클릭합니다.

3.  **🔍 변경 내용 확인**
    -   'Version Compare' 페이지로 이동하며, 두 버전 간의 차이점이 시각적으로 표시됩니다.
    -   **녹색(+):** 추가된 내용
    -   **노란색(~):** 수정된 내용
    -   **붉은색(-):** 삭제된 내용

이제 이 내용을 바탕으로 `USER_GUIDE.md` 파일을 생성하겠습니다.
